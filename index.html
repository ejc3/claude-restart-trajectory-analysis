Analysis

Token Usage Variance

Same conversation, 36% difference in tokens. Claude Code dynamically manages context based on relevance.

  • Before: 151,897 tokens with full file embeddings
  • After: 96,859 tokens with summarized content

System Reminder Accumulation

Both have <system-reminder> tags, but the pre-restart trajectory has many more:

  • 4 background Bash process notifications
  • TodoWrite tool reminders
  • Malware analysis guidance

Embedded File Content

Major contributor to bloat: Read tool results.

  • Before: Full _index.tsx (~131KB)
  • After: Content omitted or summarized

Conclusions

  1. Dynamic Context: Payloads vary significantly within the same session.
  2. System Overhead: Background processes and reminders compound over time.
  3. File Embedding: Large file reads can increase tokens by 50%+.
  4. Context Pruning: Claude Code prunes content to manage limits.
Tip: For long sessions, periodic restarts reduce cost. Be mindful of reading large files.
